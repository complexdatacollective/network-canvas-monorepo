# Remaining CSS in `apps/architect-vite`

Inventory of every `.css` file under `apps/architect-vite/src` that is still authored
as hand-written CSS (i.e. not generated by Tailwind via utilities on JSX).

Stylesheets are loaded by `apps/architect-vite/index.html`:

```html
<link rel="stylesheet" href="/src/styles/tailwind.css" />
<link rel="stylesheet" href="/src/styles/main.css" />
```

No CSS files are imported from `.tsx`/`.ts` modules (other than
`mapbox-gl/dist/mapbox-gl.css` in `MapView.tsx`, which is third-party).

> **Dead-code removal pass (done):** `home.css` (352 lines), `assets.css`
> (34 lines), and the entire orphaned `src/images/home/` directory
> (7 SVGs — `create-button.svg`, `log-out.svg`, `open-button.svg`,

> `undraw_prototyping_process.svg`, `wave.svg`) have been deleted, and

## 1. Tailwind source / design-system file (expected to stay)

### `src/styles/tailwind.css` — 870 lines
Tailwind v4 entry point. Contents are intentionally not Tailwind utilities:

- `@import "tailwindcss"` + `@source` for `@codaco/fresco-ui`.
- `@layer base` — `:root` design tokens (colors, spacing, type scale, z-index,
  picker sizes, animation timings, node/edge/ord color sequences) and base
  element styles (`html`, `body`, `strong`, `em`, `h1`–`h4`, `p`).
- `@theme static` — Tailwind v4 theme tokens (color/space/text/leading/font/
  weight, semantic colors, shadow scale, border-radius scale, keyframes).
- `@utility` blocks (Tailwind v4 custom utilities):
  `focusable`, `h1`–`h4`, `hero`, `lead`, `hint`, `code`, `action-link`,
  `nav-link`, `small-heading`, `allow-text-selection`,
  `prevent-text-selection`, `scrollable`, `form-field`, `section-container`,
  `clickable`, `clickable-2`, `clickable-3`,
  `mini-table`, `mini-table-wide`, `mini-table-rotated`, `mini-table-lite`.
- `@layer components` — vendor overrides only:
  - `.react-grid-item > .react-resizable-handle` (resize chevron).
  - `.tippy-box[data-theme~="error"]` rules (validation tooltip — Tippy
    library CSS is otherwise never imported).

> This file is the design system and is **not** a migration leftover — it is
> the supported way to declare tokens / custom utilities in Tailwind v4.

## 3. Unmigrated component CSS (the real leftovers)

Selectors below were confirmed in use by `className=` references in TSX.

### `src/styles/components/form/fields/select.css` — 162 lines — used
All `.form-fields-select*` classes for the `react-select` wrapper.
Consumers: `src/components/Form/Fields/Select.tsx` (passes
`classNamePrefix="form-fields-select"` to react-select, plus the `--has-error`
modifier) and `src/components/sections/fields/SkipLogicFields.tsx`.
Covers control, menu, option (focused/selected/active), placeholder, label,
error states (`--has-error`, `--light-background`), item add/delete/warning
icons.

### `src/lib/ProtocolSummary/styles/protocol-summary.css` — 104 lines — used
Print-specific rules that **cannot** be expressed as Tailwind utilities on
JSX. Used by `SummaryPage.tsx`, `AssetManifest.tsx`, `Entity.tsx`,
`Stage.tsx`, and `VariablePill.tsx`. Documented at the top of the file:

- `@page { margin: var(--margin) }`.
- `@media print` / `@media screen` body overrides for `.print` mode.
- `.print .protocol-summary`, `.protocol-summary-page`, `.protocol-summary`
  page-size variables.
- `.protocol-summary .page-break-marker` (screen mock-page + print
  page-break).
- `.protocol-summary [data-print="no"] { display: none }`.
- `.protocol-summary .variable-pill` — cross-area cascade into the
  `VariablePill` BEM marker (the marker was deliberately preserved for
  this hook; see `VariablePill.tsx:41-46`).

> Likely permanent — `@page` and `@media print` cannot live on a className.

### `src/styles/components/type-editor/icon-option.css` — 13 lines — used
`.type-editor-icon-option` + nested `.icon` positioning rule. Consumed by
`src/components/TypeEditor/IconOption.tsx:12`.

### `src/lib/legacy-ui/styles/components/icons.css` — 1511 lines — used
**Largest remaining file.** Per-icon SVG color rules. Top of the file
declares it is a migration-in-progress:

> "MIGRATION NOTICE: Icon colors are being migrated from SASS to Tailwind
> CSS. The new color mapping is located in `_icons-tailwind.css`. Legacy
> SASS color modifiers (`.icon--{color}`) are still supported but will be
> removed once the migration is complete."

Contains `--icon-dimension-base` variables and per-icon `&[name="…"]`
overrides with `.cls-1` … `.cls-5` SVG path fills for the legacy icon
component.

### Suggested order of attack

1. **`legacy-ui/styles/components/icons.css` (1511)** — already has a
   stated migration target (`_icons-tailwind.css`). Biggest remaining win,
   but heavy — every consumer uses `.icon`/`.cls-N` for SVG fills.
2. **`styles/components/form/fields/select.css` (162)** — move the
   `react-select` styles to the `classNames`/`styles` props on the
   React-Select instance instead of global selectors.
3. **`type-editor/icon-option.css` (13)** — small, mechanical.
4. **Leave `ProtocolSummary/styles/protocol-summary.css` (104)** — `@page`
   and `@media print` rules cannot move to JSX classNames; fold what can be
   folded into `tailwind.css` (the file's own comment notes this is the
   plan for "slice 35").
5. **Collapse the five `@import`-only barrel files** once their contents
   no longer exist.
