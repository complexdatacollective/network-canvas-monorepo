Network Canvas is a suite of free, open-source tools for surveying networks,
designed around the needs of both researchers and their participants.
We provide a complete end-to-end workflow for networks research, with apps
for survey design and for interviewing. Using these tools, researchers can easily design, capture, and export network data.

## Getting Started

To get started using the Network Canvas tools, read the
[project overview](/en/project/project-information/project-overview), and determine which tools tools
to use for your project.

|                      | Runs where            | Access                                                               | Step in workflow | Supported Schemas |
| -------------------- | --------------------- | -------------------------------------------------------------------- | ---------------- | ----------------- |
| Architect Web        | Browser               | [architect.networkcanvas.com](architect.networkcanvas.com)           | Build protocol   | V8                |
| Architect Desktop    | Windows, macOS, Linux | [Download](/en/project/getting-started/downloading-desktop-software) | Build protocol   | V7                |
| Interviewer 6.x      | Desktop & tablet      | [Download](/en/project/getting-started/downloading-desktop-software) | Run interviews   | V7                |
| Interviewer 7 (beta) | Desktop & tablet      | [Download](/en/project/getting-started/downloading-desktop-software) | Run interviews   | V8                |
| Fresco               | Browser (self-hosted) | [Deploy an instance](/en/run-interview/fresco/deployment/guide)      | Run interviews   | V8                |

<TipBox>

Not all apps are compatible. A protocol's schema version, determined by the tool it was
built in, controls which apps can open it, so consider compatibility when choosing
your tools. See [Protocol compatibility](/en/project/getting-started/protocol-compatibility)
for a visual matrix of what runs where, or
[Protocol Schema Information](/en/project/advanced-topics/protocol-schema-information) for a
full explanation of schema versions and automatic migration.

</TipBox>

From there, explore the two main sections of the documentation, each with step-by-step
tutorials to get you started:

- **[Build Protocols](/en/build-protocol)** — everything about designing a protocol in
  Architect. Choose between **Architect Web** and **Architect Desktop**, open or install
  the tool, and follow the tutorials for building a protocol.
- **[Run Interviews](/en/run-interview)** — everything about administering interviews.
  Choose between **Interviewer** — the desktop and tablet application — and **Fresco**,
  which deploys your protocol to the web.

We encourage you to use the search function at the top of each page to find articles on
any given topic. Our pages are heavily interlinked, and information is necessarily spread
around.

## Useful Pages

Please explore the pages and headings to the left.

This **project** section holds the supporting reference material that applies across all
of the tools. You'll find these subsections in the sidebar to the left:

- **Getting Started** — contains articles that cover essential basic information, such as
  how to install the software.
- **Tutorials** — in-depth, end-to-end walkthroughs that span the
  whole workflow. Tutorials specific to building protocols or running interviews live in the
  [Build Protocols](/en/build-protocol) and [Run Interviews](/en/run-interview) sections.
- **Project Information** — contains details of the origins of the projects, as well as
  information about security, IRB, and collaborating with our projects.
- **Advanced Topics** — contains articles that may be useful for advanced users or
  developers looking to implement new features.
- **Interface Documentation** — if you are looking for specific information about the
  configuration options of an interface, this section contains a page dedicated to each
  type.
- **Key Concepts** — which has self-contained articles on all the fundamental concepts
  within the Network Canvas software.

Whichever tools you choose, the shape of a study is the same: design a protocol, run it
to collect data, and export that data for analysis. For a detailed walkthrough of one such
workflow — creating a protocol in Architect Desktop and deploying to Interviewer — see the
[protocol and data workflows](/en/project/tutorials/protocol-and-data-workflows) tutorial.
