## What is Architect Web?

Architect Web is the browser-based version of Architect, the protocol designer for Network Canvas. It runs entirely in your web browser — there is nothing to download or install — and lets you compose name generators, capture ordinal and categorical data, map connections, and explore narratives, just as you can in the desktop application.

Architect Web and Architect Desktop share the same interface and protocol-building workflow, but they are **not interchangeable**: Architect Web builds protocols on the latest schema (version 8), for running interviews in [Fresco](/en/run-interview/fresco/about), while Architect Desktop builds on schema 7, for running interviews in the desktop and tablet [Interviewer](/en/run-interview/interviewer) app. Which you should use depends on where you intend to run your interviews — see the comparison in [About Architect](/en/build-protocol) and the [protocol schema information](/en/project/advanced-topics/protocol-schema-information) article for full details.

## Opening Architect Web

Architect Web is available at [architect.networkcanvas.com](https://architect.networkcanvas.com) — open it in any modern web browser; there is nothing to install.

When you open Architect Web, you are greeted by the **home screen**. A welcome panel introduces the app, with links to the documentation, community, and code repository. From here you have two main tasks: create a new protocol, or open an existing one.

![The Architect Web home screen.](/assets/img/architect-web/home-screen.png)

## Creating a new protocol

To build a protocol from scratch, click **Create a new protocol**. A dialog appears asking you to give your protocol a name — a name is required before you can continue. Enter a name and click **Create Protocol**. Your new, empty protocol opens in the editor.

![The new protocol dialog.](/assets/img/architect-web/new-protocol-dialog.png)

## Opening an existing protocol

If you already have a protocol saved as a `.netcanvas` file, click **Open existing protocol** and choose the file, or simply drag and drop a `.netcanvas` file anywhere on the home screen.

## Your protocol library

You can also reopen protocols you have worked on previously, directly from the home screen. Architect Web keeps a **library** of your protocols, saved in this browser:

- The **Recent** tab lists the protocols saved in this browser, showing how many stages each one has and when it was added and last edited. Click a protocol to open it.
- The **Templates** tab includes a **Sample Protocol** — a good starting point if this is your first time using Architect. Select **Use this template**, give your copy a name, and it opens ready to edit.

![The Recent and Templates tabs on the home screen.](/assets/img/architect-web/library-tabs.png)

Because this library lives in your browser rather than on a server, it is important to understand what that means for backing up and moving your work. See [Storing your work](/en/build-protocol/architect-web/storing-your-work).

## The protocol overview screen

Once you have created or opened a protocol, you are taken to the **protocol overview screen**. At the top, a card shows your protocol's name and description — both of which you can edit here — along with a count of its stages, node types, and edge types.

Below the card is the **timeline**, which represents the stages of your interview in the sequence that your participants will see them. Since you have just created a new protocol, the timeline is empty apart from a button that enables you to add your first stage.

![The protocol overview screen with an empty timeline.](/assets/img/architect-web/overview-screen.png)

At any time, you can use **Download** to export your protocol as a `.netcanvas` file — for example to back it up, or to upload it to [Fresco](/en/run-interview/fresco/using-fresco#uploading-protocols) to run interviews.
