## Do I need to install anything to use Architect Web?

No. Architect Web runs entirely in your web browser. Open [architect.networkcanvas.com](https://architect.networkcanvas.com) and start building — there is nothing to download or install.

## Where is my work saved? Is it on a server?

Your protocols are saved in a **library** inside your web browser, on the device you are using. They are never uploaded to a server. See [Storing your work](/en/build-protocol/architect-web/storing-your-work) for what this means for backing up and moving protocols.

## Do I have to save my work manually?

No. Architect Web saves automatically as you edit — there is no **Save** button. Your changes are written continuously to the browser's library.

## Will I lose my protocols if I clear my browser data?

Yes. Because your protocols are stored in the browser, clearing your browsing data for the site will permanently remove them, as will using **Clear all protocols** on the home screen. To keep a copy you won't lose, **Download** your protocol as a `.netcanvas` file. See [What removes your protocols](/en/build-protocol/architect-web/storing-your-work#what-removes-your-protocols).

## Can I open my protocol on a different computer or in a different browser?

Not directly — the library is tied to the specific browser and device where you created the protocol. To move your work, **Download** the protocol as a `.netcanvas` file and open that file from the home screen on the other browser or device.

## Is there a limit to how much I can store?

Browsers limit how much data a site may store, and large protocols — especially those with many or large media assets — use more of that allowance. Browsers may also evict this kind of local data under storage pressure. For anything you cannot afford to lose, keep a downloaded `.netcanvas` copy. See [Storage limits and eviction](/en/build-protocol/architect-web/storing-your-work#storage-limits-and-eviction).

## Can I undo a change I didn't mean to make?

Yes. Use **Undo** and **Redo** in the editor toolbar to step backward and forward through your recent edits within the current session.

## Which browsers does Architect Web support?

Architect Web works in any modern, up-to-date web browser, such as recent versions of Chrome, Edge, Firefox, and Safari. For the best experience, keep your browser updated.

## Should I use Architect Web or Architect Desktop?

They are **not interchangeable**, and they do not produce identical protocols. Architect Web builds protocols on the latest protocol schema (version 8); Architect Desktop builds on schema 7 and earlier. Which you should use depends on where you intend to run your interviews:

- Choose **Architect Web** if you will run interviews in **Fresco**. Schema 8 — and the newest interfaces, such as Geospatial — is supported only by Architect Web and Fresco.
- Choose **Architect Desktop** if you will run interviews in the desktop or tablet **Interviewer** app, which cannot run schema 8 protocols. Protocols for Interviewer must be built and kept on schema 7 in Architect Desktop.

Architect Web can open a schema 7 protocol made in Architect Desktop and upgrade it to schema 8, but this is one-way — a schema 8 protocol cannot be converted back or opened in Architect Desktop. See [Protocol Schema Information](/en/project/advanced-topics/protocol-schema-information) for full details.

## Can I run a protocol I built in Architect Web in Fresco or Interviewer?

In **Fresco**, yes: **Download** your protocol as a `.netcanvas` file and upload it to [Fresco](/en/run-interview/fresco/using-fresco#uploading-protocols). Protocols built in Architect Web use schema 8, which runs in Fresco. They **cannot** be run in the desktop or tablet **Interviewer** app, which does not support schema 8 — to collect data in Interviewer, build your protocol in [Architect Desktop](/en/build-protocol/architect-desktop) instead.

## Is Architect Web's protocol compatible with Architect Desktop?

Not in both directions. Architect Web saves protocols on the latest schema (version 8), which **cannot** be opened in Architect Desktop or run in the desktop or tablet Interviewer app — schema 8 is supported only by Architect Web and Fresco. Architect Web can, however, open a protocol made in Architect Desktop (schema 7 or earlier) and automatically upgrade it to schema 8. Because that upgrade is one-way, keep a copy of your original schema 7 file if you still need it in Architect Desktop or Interviewer. See [Protocol Schema Information](/en/project/advanced-topics/protocol-schema-information).
