Network Canvas is a suite of free, open-source tools for surveying networks,
designed around the needs of both researchers and their participants.
We provide a complete end-to-end workflow for networks research, with apps
for survey design and for interviewing. Using these tools, researchers can easily design, capture, and export network data.

## Getting Started

To get started using the Network Canvas tools, read the
[project overview](/en/get-started/project-information/project-overview), and determine which tools
to use for your project.

|                      | Runs where            | Access                                                             | Step in workflow | Supported Schemas |
| -------------------- | --------------------- | ------------------------------------------------------------------ | ---------------- | ----------------- |
| Architect Web        | Browser               | [architect.networkcanvas.com](https://architect.networkcanvas.com) | Build protocol   | V8                |
| Architect Desktop    | Windows, macOS, Linux | [Download](/en/design-protocols/installing-architect-desktop)      | Build protocol   | V7                |
| Interviewer 6.x      | Desktop & tablet      | [Download](/en/collect-data/interviewer/installing-interviewer)    | Run interviews   | V7                |
| Interviewer 8 (beta) | Desktop & tablet      | [Download](/en/collect-data/interviewer/installing-interviewer)    | Run interviews   | V8                |
| Fresco               | Browser (self-hosted) | [Deploy an instance](/en/collect-data/fresco/guide)                | Run interviews   | V8                |

<TipBox>

Not all apps are compatible. A protocol's schema version, determined by the tool it was
built in, controls which apps can open it, so consider compatibility when choosing
your tools. See [App compatibility by schema version](/en/get-started/release-announcements)
for a visual matrix of what runs where, or
[Protocol Schema Information](/en/get-started/advanced-topics/protocol-schema-information) for a
full explanation of schema versions and automatic migration.

</TipBox>

From there, explore the two main sections of the documentation, each with step-by-step
tutorials to get you started:

- **[Design Protocols](/en/design-protocols)** — everything about designing a protocol in
  Architect. Choose between **Architect Web** and **Architect Desktop**, open or install
  the tool, and follow the tutorials for building a protocol.
- **[Collect Data](/en/collect-data)** — everything about administering interviews.
  Choose between **Interviewer** — the desktop and tablet application — and **Fresco**,
  which deploys your protocol to the web.

We encourage you to use the search function at the top of each page to find articles on
any given topic. Our pages are heavily interlinked, and information is necessarily spread
around.

## Useful Pages

Please explore the pages and headings to the left.

This **Get Started** section holds the supporting reference material that applies across
all of the tools. You'll find these headings in the sidebar to the left:

- **Planning a Study** — how to think through your study before you start, including a
  visual [overview of the workflows](/en/get-started/planning-a-study/workflows) and
  guidance on GDPR and IRB. The full end-to-end
  [Protocol and Data Workflows](/en/collect-data/protocol-and-data-workflows) tutorial
  lives in the [Collect Data](/en/collect-data) section.
- **Project Information** — details of the origins of the projects, as well as information
  about the [security model](/en/get-started/project-information/security-model), citing
  the software, and collaborating with us.
- **Advanced Topics** — articles that may be useful for advanced users or developers,
  including protocol schema and file-format details.

You'll also find [release announcements](/en/get-started/release-announcements) here.
Reference material for _designing_ protocols — **Key Concepts** and **Interface
Documentation** — now lives in the [Design Protocols](/en/design-protocols) section.

Whichever tools you choose, the shape of a study is the same: design a protocol, run it
to collect data, and export that data for analysis. For a detailed walkthrough of one such
workflow — creating a protocol in Architect Desktop and deploying to Interviewer — see the
[protocol and data workflows](/en/collect-data/protocol-and-data-workflows) tutorial.
