title: Collecting Data in Network Canvas
navOrder: 0
---

With your protocol built, this phase of your study is about running interviews and gathering data. Depending on your design, interviews can happen in person on a device you control, or remotely in participants' own browsers. Alongside the interviews themselves, you will also manage the practical side of fieldwork: getting protocols onto devices, and getting interview data back off them.

The central decision at this stage is which app to collect data with. Network Canvas offers two: **Interviewer**, the browser-based app — installable to a device you control and usable offline — for in-person, interviewer-assisted interviews, and **Fresco**, which lets participants complete interviews remotely in a web browser. The main difference is where the interview happens and who is in control. Which app can run a given protocol also depends on its schema version — see [Protocol Schema Information](/en/get-started/advanced-topics/protocol-schema-information). The table below compares them.

<KeyConcept title="Not sure which to use?">

Choose **Interviewer** for in-person interviews guided by a trained interviewer on a device you control.

Choose **Fresco** if you need participants to complete interviews remotely, in their own browser, and you can host the app yourself.

Interviewer itself comes in two supported versions: the normal app, which runs in your web browser — installable to a device you control — with an improved interview experience and support for schema 8, and **Interviewer Classic**, the established offline desktop and tablet app you download and install (schema 7). Interviewer is recommended for all new studies.

</KeyConcept>

|                                   | [Interviewer](/en/collect-data/interviewer)                                                                                                                                                         | [Fresco](/en/collect-data/fresco/about)                                          |
| --------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| Runs on                           | Interviewer runs in a web browser (installable as a PWA); Interviewer Classic is a desktop & tablet app                                                                                             | Web browser (desktop or laptop — not mobile)                                     |
| Setup                             | Open [Interviewer](/en/collect-data/interviewer) in your browser — or [download and install](/en/collect-data/interviewer/installing-interviewer) Interviewer Classic on each device                | [Deploy and host your own instance](/en/collect-data/fresco/guide)               |
| How interviews happen             | In person, guided by a trained interviewer on a controlled device                                                                                                                                   | Remotely — participants visit a URL and complete it themselves                   |
| Connectivity                      | Interviewer Classic works fully offline; Interviewer is offline-capable, but some schema-8 interfaces (e.g. [Geospatial](/en/design-protocols/interface-documentation/geospatial)) require internet | Requires internet; runs on a server                                              |
| Where data lives                  | On the device, exported as files you transfer yourself                                                                                                                                              | In a database on your server, exported from the dashboard                        |
| Protocol & participant management | Per device, via file or URL import                                                                                                                                                                  | Centralized web dashboard                                                        |
| Best for                          | In-person, interviewer-assisted studies; offline settings                                                                                                                                           | Remote data collection and centralized study management                          |
| Status                            | Interviewer and Interviewer Classic supported                                                                                                                                                       | Supported ([known limitations](/en/collect-data/fresco/about#known-limitations)) |

<TipBox>

A **PWA** (Progressive Web App) is a website you can optionally install to your device. Interviewer runs in any modern web browser with nothing to install; installing it as a PWA adds quick access from your device and offline use.

</TipBox>

Once you have chosen an app, the [Interviewer](/en/collect-data/interviewer) and [Fresco](/en/collect-data/fresco/about) pages cover each one in detail. When you are ready to move protocols onto devices and bring interview data back, [Protocol and Data Workflows](/en/collect-data/protocol-and-data-workflows) walks through the whole process end to end.
