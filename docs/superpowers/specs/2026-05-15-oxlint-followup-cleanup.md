# oxlint follow-up cleanup

After migrating from Biome to oxlint + oxfmt, the new tooling surfaced 1,448
lint errors and 2,158 warnings that the previous Biome configuration did not
flag (or flagged at a different severity). The migration PR resolved the
errors needed to make `pnpm lint` exit 0 by:

1. Fixing inline issues that were small in number and mechanical (~36 inline
   fixes across unused type imports, `new Array(n)` patterns, sort comparators,
   and the like).
2. Adjusting the `no-unused-vars` config to add `ignoreRestSiblings`,
   `varsIgnorePattern`, and `destructuredArrayIgnorePattern` for the React
   props-spread idiom.
3. Scoping a few rules off in specific directories (Storybook stories,
   Playwright e2e fixtures, build scripts, polyfill shims, test mocks,
   the architect-desktop `development-protocol/` template directory).
4. Downgrading a number of rules from `error` to `warn` where the error count
   was large and either (a) the rule was newly strict compared to Biome, or
   (b) the rule was producing a high rate of type-aware false positives in
   `.js` files where the type information is insufficient.

This document records the rules that were downgraded or scoped so that future
cleanup can promote them back to `error` once the underlying code has been
fixed.

## Rules scoped off in specific directories

These are intentional carve-outs, not deferred work:

- `react/rules-of-hooks` is off in `**/*.stories.{ts,tsx,js,jsx}` and `**/.storybook/**` — Storybook `render: () => {...}` callbacks are not React components, but the rule cannot distinguish them.
- `react/rules-of-hooks` is off in `packages/interview/e2e/**` — Playwright fixtures use a `use` callback which the rule misidentifies as a React hook.
- `react/rules-of-hooks` is off in `**/scripts/**` — build/release scripts that happen to define a function starting with a lowercase letter.
- `no-process-env` and `no-console` remain off in `**/scripts/**`, `**/vite.config.*`, `**/vitest.config.*`, `**/next.config.*`, `tooling/**`, and `apps/architect-desktop/**` — legitimate uses.
- `no-unused-vars` is off in `apps/architect-desktop/development-protocol/**` — these files are protocol templates with documented function signatures whose parameters are referenced from the protocol runtime, not from the source itself.
- `no-unused-expressions` is off in `apps/architect-desktop/src/utils/webShims/**` — polyfill shims using the `condition || (assignment)` IIFE idiom.
- `no-import-assign` is off in `**/__tests__/**` and `**/*.test.*` — test files commonly reassign imported helpers for mocking.
- `typescript/unbound-method` is off in tests (existing carve-out).

## Suggested order for cleanup

1. **High-value type-aware false positives**: revisit `no-redundant-type-constituents`, `await-thenable`, `restrict-template-expressions`, `no-implied-eval`, `unbound-method` once we understand why oxlint cannot resolve workspace types from these call sites. This is likely a tsconfig path / project-reference issue and might be fixable in one pass.
2. **A11y audit**: address `jsx-a11y/click-events-have-key-events`, `no-static-element-interactions`, `prefer-tag-over-role`, `no-autofocus`, and `media-has-caption` together. There is enough repeated structure to justify a single sweep.
3. **Tailwind class allowlist**: configure `tailwindcss/no-unknown-classes` to know about the project's custom utility classes (`form-field-container`, `page-break-marker`, etc.) and the shadcn animation classes, or refactor those usages.
4. **React legacy migration**: address `react/no-unsafe` and `react/no-find-dom-node` together by converting the remaining class components to hooks.
5. **Switch exhaustiveness** and **no-floating-promises**: per-site audits.
6. **Pre-existing import cycles**: untangle each cycle by extracting types.
