# Legacy Interviewer — Capacitor Mobile Capability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restore iPadOS/Android support for the legacy Network Canvas Interviewer (`apps/interviewer`) by replacing the dead Cordova adapter layer with Capacitor, adding a web build, and committing native iOS/Android projects.

**Architecture:** The renderer already switches platform behaviour through `inEnvironment(tree)` (`ELECTRON | CORDOVA | WEB`). We rename `CORDOVA` → `CAPACITOR` and reimplement each adapter branch with `@capacitor/*` plugins, keeping every exported signature unchanged so renderer call sites are untouched. A new `vite.web.config.js` produces a static web bundle (`dist-web/`) that a Capacitor shell (committed `ios/` + `android/`) loads. Native calls are gated by `Capacitor.isNativePlatform()`; a plain browser degrades to the existing `WEB` behaviour.

**Tech Stack:** Capacitor 8 (`@capacitor/core|ios|android|filesystem|share|app|device|browser`), `@capawesome/capacitor-file-picker`, `localforage` (IndexedDB redux-persist engine), Vite (web build), Vitest (tests). Reference implementation: `apps/interviewer-v7`.

**Spec:** `docs/superpowers/specs/2026-06-08-interviewer-capacitor-mobile-design.md`

**Path strategy:** Capacitor's Filesystem is `{ path, directory }`-based. The app concatenates `userDataPath()`/`tempDataPath()` prefixes onto relative paths. For Capacitor, `userDataPath()` returns `''` (root of `Directory.Data`) and `tempDataPath()` returns `'tmp/'` (a folder under `Directory.Data`); everything lives under `Directory.Data`. `capacitorPath(urlString)` strips a leading slash and returns `{ directory: Directory.Data, path }`.
