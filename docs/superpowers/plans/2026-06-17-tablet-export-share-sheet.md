# Tablet Export Share Sheet Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** On the Capacitor (iPadOS / Android) build, exporting interview data presents the native OS share sheet (with "Save to Files" location picker) instead of silently writing the zip to `Documents/`.

**Architecture:** Change only the Capacitor branch of `downloadBlob` in `apps/interviewer-v8/src/lib/files/download.ts`. After building the zip, write it to the ephemeral `Directory.Cache`, present the share sheet via `@capacitor/share`, then best-effort delete the cache file. The `downloadBlob` signature and the `DownloadResult` (`{ saved, path? }`) contract are unchanged, so the sole caller (`useSessionMutations.handleExport`) and its toasts/`markSessionsExported` need no edits. Electron and web branches are untouched.

**Tech Stack:** TypeScript, React, Capacitor 8 (`@capacitor/share`, `@capacitor/filesystem`), Vitest (jsdom), oxlint + oxfmt, pnpm workspaces.

## Global Constraints

- No `any` types; no `as` assertions to bypass type checking — fix the underlying type instead.
- No barrel files; no re-exports for convenience. Import from the original source.
- Formatting/lint: 2-space indentation, single quotes (oxfmt/oxlint). Run `pnpm lint:fix` from the repo root before committing; use `--fix` for autofixable issues.
- Only comment code where the usage is unusual or complex.
- `@capacitor/share` must be on the **v8 line**, matching the rest of the Capacitor 8 stack (`@capacitor/core` `^8.4.0`, `@capacitor/filesystem` `^8.1.2`, etc.).
- Only the Capacitor branch of `downloadBlob` changes. Do **not** alter the `downloadBlob` signature, the `DownloadResult` type, the Electron branch, the web branch, or `useSessionMutations`.
- Tests are co-located in `__tests__/` with `.test.ts` extensions, using Vitest. Unit tests run under the `unit` project (jsdom). Run them with `pnpm --filter @codaco/interviewer-v8 test`.
- **Native registration is out of scope for this plan.** Do NOT run `cap sync`, `pod install`, or edit `ios/App/Podfile` or `android/capacitor.settings.gradle`. The working tree already has unrelated uncommitted native changes; `@capacitor/share` will be picked up automatically on the maintainer's next `cap sync`. Only stage the specific files each task names — never `git add -A` / `git add .` (there are untracked files and uncommitted native files in the tree that must not be swept into commits).

- [ ] **Step 3: Implement the share-sheet branch**

In `apps/interviewer-v8/src/lib/files/download.ts`, replace the existing `isCapacitor` block (currently writing to `Directory.Documents`) with the version below. Leave the Electron branch, the web branch, the `downloadBlob` signature, the `DownloadResult` type, and `blobToBase64` unchanged.

```ts
if (isCapacitor) {
  const { Filesystem, Directory } = await import('@capacitor/filesystem');
  const { Share } = await import('@capacitor/share');
  const base64 = await blobToBase64(blob);
  // Write to Cache (ephemeral, shareable) rather than Documents so we leave no
  // copy behind; the share sheet lets the user choose the real destination.
  const { uri } = await Filesystem.writeFile({
    path: suggestedName,
    data: base64,
    directory: Directory.Cache,
    recursive: true,
  });
  try {
    const { value: canShare } = await Share.canShare();
    if (!canShare) {
      throw new Error('Sharing is not available on this device');
    }
    await Share.share({ title: suggestedName, files: [uri] });
    return { saved: true };
  } catch (cause) {
    if (isShareCanceled(cause)) {
      return { saved: false };
    }
    throw cause;
  } finally {
    try {
      await Filesystem.deleteFile({
        path: suggestedName,
        directory: Directory.Cache,
      });
    } catch {
      // Best-effort cleanup; a leftover cache entry must not mask the result.
    }
  }
}
```

Then add this helper below `blobToBase64` (at the bottom of the file):

```ts
// `@capacitor/share` rejects with a "share canceled" message when the user
// dismisses the sheet (iOS); treat that as a non-error cancellation.
function isShareCanceled(cause: unknown): boolean {
  return cause instanceof Error && /cancel/i.test(cause.message);
}
```
